Of course. Here is a complete design and implementation for an asynchronous file processing service.

### System Architecture

The service is designed using a microservices architecture to decouple the web-facing API from the background processing task.

1.  **API Service (`api`)**: A Python FastAPI application that handles HTTP requests. Its sole responsibilities are:
    *   Accepting `POST /upload` requests with a file.
    *   Creating a job entry in the PostgreSQL database with a `QUEUED` status.
    *   Saving the uploaded file to a shared volume.
    *   Publishing a message with the `job_id` to a RabbitMQ queue.
    *   Responding immediately to the client with the `job_id`.
    *   Handling `GET /status/{job_id}` requests to fetch job status from the database.

2.  **Worker Service (`worker`)**: A Python script that acts as a consumer.
    *   Listens for messages on the RabbitMQ queue.
    *   When a message is received, it updates the job status to `PROCESSING`.
    *   It reads the corresponding file from the shared volume.
    *   It uses the `hachoir` library to extract metadata from the file.
    *   It updates the job record in PostgreSQL with the extracted metadata and sets the status to `COMPLETED` or `FAILED`.

3.  **PostgreSQL (`postgres`)**: The relational database for storing job information, including status and the final metadata (stored in a `JSONB` column).

4.  **RabbitMQ (`rabbitmq`)**: A message broker that queues jobs, ensuring that file uploads can be accepted quickly without waiting for processing to complete.

5.  **Shared Volume (`uploads`)**: A Docker volume mounted by both the `api` and `worker` services, allowing the `api` to write files and the `worker` to read them.

This architecture is robust, scalable, and resilient. If the worker fails, the message remains in the queue (or is re-queued) to be processed later.

```python
import uuid
import pika
import shutil
from pathlib import Path
from datetime import datetime
from typing import Optional, Dict, Any

from fastapi import FastAPI, UploadFile, File, HTTPException, Depends, Request
from sqlalchemy.ext.asyncio import AsyncSession
from sqlalchemy.future import select
from pydantic import BaseModel, Field

from shared.database import get_db, engine
from shared.models import Job, Base, StatusEnum
from shared.config import settings

# --- Pydantic Models for API responses ---

class JobCreatedResponse(BaseModel):
    job_id: uuid.UUID
    status_url: str = Field(..., example="/status/a1b2c3d4-e5f6-7890-1234-567890abcdef")

class JobStatusResponse(BaseModel):
    id: uuid.UUID
    original_filename: str
    status: StatusEnum
    metadata: Optional[Dict[str, Any]] = None
    created_at: datetime
    updated_at: datetime

    class Config:
        from_attributes = True

# --- FastAPI App Initialization ---

app = FastAPI(
    title="File Metadata Extraction Service",
    description="An API to upload files for asynchronous metadata extraction.",
    version="1.0.0"
)

# --- Database Initialization ---

@app.on_event("startup")
async def startup():
    async with engine.begin() as conn:
        # For a real app, use Alembic migrations instead of create_all
        await conn.run_sync(Base.metadata.create_all)
    Path(settings.UPLOAD_DIR).mkdir(parents=True, exist_ok=True)


# --- RabbitMQ Connection ---
def get_rabbitmq_connection():
    return pika.BlockingConnection(pika.URLParameters(settings.RABBITMQ_URL))

# --- API Endpoints ---

@app.post("/upload", response_model=JobCreatedResponse, status_code=202)
async def upload_file(
    request: Request,
    file: UploadFile = File(...),
    db: AsyncSession = Depends(get_db)
):
    job_id = uuid.uuid4()
    file_extension = Path(file.filename).suffix
    saved_filename = f"{job_id}{file_extension}"
    file_path = Path(settings.UPLOAD_DIR) / saved_filename

    # Save the file to the shared volume
    try:
        with file_path.open("wb") as buffer:
            shutil.copyfileobj(file.file, buffer)
    finally:
        file.file.close()

    # Create a job record in the database
    new_job = Job(
        id=job_id,
        original_filename=file.filename,
        status=StatusEnum.QUEUED
    )
    db.add(new_job)
    await db.commit()
    await db.refresh(new_job)

    # Publish job_id to RabbitMQ
    try:
        connection = get_rabbitmq_connection()
        channel = connection.channel()
        channel.queue_declare(queue=settings.RABBITMQ_QUEUE, durable=True)
        channel.basic_publish(
            exchange='',
            routing_key=settings.RABBITMQ_QUEUE,
            body=str(job_id),
            properties=pika.BasicProperties(delivery_mode=pika.DeliveryMode.Persistent)
        )
        connection.close()
    except Exception as e:
        # If queuing fails, mark job as failed and raise an error
        new_job.status = StatusEnum.FAILED
        await db.commit()
        raise HTTPException(status_code=500, detail=f"Could not queue job: {e}")

    return {
        "job_id": job_id,
        "status_url": request.url_for('get_status', job_id=str(job_id))
    }


@app.get("/status/{job_id}", response_model=JobStatusResponse)
async def get_status(job_id: uuid.UUID, db: AsyncSession = Depends(get_db)):
    result = await db.execute(select(Job).where(Job.id == job_id))
    job = result.scalar_one_or_none()

    if job is None:
        raise HTTPException(status_code=404, detail="Job not found")

    return job
```

**`src/worker/main.py`**
```python
import uuid
import pika
import time
import json
from pathlib import Path
from hachoir.parser import createParser
from hachoir.metadata import extractMetadata

from sqlalchemy.ext.asyncio import create_async_engine, AsyncSession
from sqlalchemy.orm import sessionmaker
from sqlalchemy.future import select

from shared.models import Job, StatusEnum
from shared.config import settings

# --- Database Setup (separate from FastAPI's) ---
engine = create_async_engine(settings.DATABASE_URL)
AsyncSessionLocal = sessionmaker(engine, class_=AsyncSession, expire_on_commit=False)

# --- Metadata Extraction Logic ---
def extract_file_metadata(file_path: Path) -> dict:
    """Extracts metadata from a file using Hachoir."""
    metadata_dict = {}
    parser = createParser(str(file_path))
    if not parser:
        return {"error": "Unable to create parser for this file type."}

    with parser:
        try:
            metadata = extractMetadata(parser)
            if not metadata:
                 return {"error": "Unable to extract metadata."}

            for key, value in metadata.exportDictionary().items():
                if isinstance(value, list) and value:
                    # Hachoir often returns lists, take the first item's text
                    metadata_dict[key] = value[0].text
                elif hasattr(value, 'text'):
                    metadata_dict[key] = value.text
                else:
                    metadata_dict[key] = str(value)

        except Exception as err:
            return {"error": f"Metadata extraction failed: {err}"}
    return metadata_dict

# --- RabbitMQ Consumer Logic ---
async def process_job(job_id_str: str):
    """The core logic for processing a single job."""
    print(f" [x] Received job ID: {job_id_str}")
    job_id = uuid.UUID(job_id_str)

    async with AsyncSessionLocal() as session:
        try:
            # 1. Fetch job and update status to PROCESSING
            result = await session.execute(select(Job).where(Job.id == job_id))
            job = result.scalar_one_or_none()
            if not job:
                print(f" [!] Job {job_id} not found in database. Ignoring.")
                return

            job.status = StatusEnum.PROCESSING
            await session.commit()
            print(f" [*] Processing job {job.id} for file {job.original_filename}")

            # 2. Find file and extract metadata
            file_extension = Path(job.original_filename).suffix
            file_path = Path(settings.UPLOAD_DIR) / f"{job.id}{file_extension}"

            if not file_path.exists():
                raise FileNotFoundError(f"File not found at {file_path}")

            metadata = extract_file_metadata(file_path)

            # 3. Update job with result
            job.metadata = metadata
            job.status = StatusEnum.COMPLETED
            print(f" [+] Job {job.id} completed successfully.")

        except Exception as e:
            print(f" [!] Error processing job {job_id}: {e}")
            if 'job' in locals() and job:
                job.status = StatusEnum.FAILED
                job.metadata = {"error": str(e)}
        finally:
            if 'job' in locals() and job:
                await session.commit()


async def rabbitmq_callback(ch, method, properties, body):
    job_id = body.decode()
    await process_job(job_id)
    ch.basic_ack(delivery_tag=method.delivery_tag)


def main():
    print(' [*] Worker starting. To exit press CTRL+C')
    
    # Simple retry logic for RabbitMQ connection
    connection = None
    for i in range(10):
        try:
            connection = pika.BlockingConnection(pika.URLParameters(settings.RABBITMQ_URL))
            print(" [+] Connected to RabbitMQ.")
            break
        except pika.exceptions.AMQPConnectionError:
            print(f" [!] RabbitMQ not available, retrying in 5 seconds... ({i+1}/10)")
            time.sleep(5)

    if not connection:
        print(" [!] Could not connect to RabbitMQ after multiple retries. Exiting.")
        return

    channel = connection.channel()
    channel.queue_declare(queue=settings.RABBITMQ_QUEUE, durable=True)
    channel.basic_qos(prefetch_count=1)

    # We need to wrap the async callback to be called from the sync pika library
    def sync_callback(ch, method, properties, body):
        import asyncio
        asyncio.run(rabbitmq_callback(ch, method, properties, body))
        
    channel.basic_consume(queue=settings.RABBITMQ_QUEUE, on_message_callback=sync_callback)
    
    try:
        channel.start_consuming()
    except KeyboardInterrupt:
        print(" [-] Worker stopping.")
        channel.stop_consuming()
    finally:
        connection.close()


if __name__ == '__main__':
    main()
```

### How to Run

1.  **Prerequisites**: Docker and Docker Compose must be installed.
2.  **Environment**: Create a `.env` file in the project root by copying `.env.example` and filling in the values (the defaults will work fine).
3.  **Build and Start**: Open a terminal in the project's root directory and run:
    ```bash
    docker-compose up --build
    ```
4.  **Test the Service**:
    *   **Upload a file** (e.g., an image named `test.jpg`):
        ```bash
        curl -X POST "http://localhost:8000/upload" \
             -F "file=@/path/to/your/test.jpg"
        ```
        You will receive a JSON response with a `job_id`.
        ```json
        {
          "job_id": "c2a2d2e1-7c9b-4f7d-8f9c-1f4a9b3d0a1b",
          "status_url": "/status/c2a2d2e1-7c9b-4f7d-8f9c-1f4a9b3d0a1b"
        }
        ```
    *   **Check the status**: Use the `job_id` from the response to check the status. It will initially be `QUEUED` or `PROCESSING`.
        ```bash
        curl http://localhost:8000/status/c2a2d2e1-7c9b-4f7d-8f9c-1f4a9b3d0a1b
        ```
    *   **View the result**: After a moment, check again. The status should be `COMPLETED` and the `metadata` field will be populated.
        ```json
        {
          "id": "c2a2d2e1-7c9b-4f7d-8f9c-1f4a9b3d0a1b",
          "original_filename": "test.jpg",
          "status": "COMPLETED",
          "metadata": {
            "Image width": "1920 pixels",
            "Image height": "1080 pixels",
            "MIME type": "image/jpeg",
            "File size": "256 KB",
            "Creation date": "2023-11-28 10:30:00"
            /* ... other metadata ... */
          },
          "created_at": "2023-11-28T18:45:00.123456",
          "updated_at": "2023-11-28T18:45:02.654321"
        }
        ```
5.  **Shutdown**: Press `CTRL+C` in the terminal where `docker-compose` is running, then run `docker-compose down` to stop and remove the containers.
